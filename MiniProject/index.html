Project Context

Using SQL to analyze transactional data from a fictional pizza store to answer critical business questions.

Phase 1: Data Inspection

Ensuring data quality and understanding the menu structure.

1. Unique Categories:

SELECT DISTINCT category FROM pizza_types;

2. Handling Nulls (Ingredients):

SELECT name, COALESCE(ingredients, 'Missing Data') 
FROM pizza_types LIMIT 5;

3. Missing Prices Check:

SELECT * FROM pizzas WHERE price IS NULL;

Phase 2: Filtering & Sorting

Daily Snapshot (Jan 1st):

SELECT order_id, date, time 
FROM orders 
WHERE date = '2015-01-01';

High Value Items:

SELECT pizza_id, price FROM pizzas 
ORDER BY price DESC;

Complex OR Logic:

SELECT * FROM orders
WHERE date = '2015-02-15' 
   OR time > '20:00:00';

Phase 2: Targeted Exploration

Price Bracket ($15-$17):

SELECT pizza_id, price FROM pizzas
WHERE price BETWEEN 15.00 AND 17.00;

Large Format (L/XL):

SELECT * FROM pizzas
WHERE size = 'L' OR size = 'XL';

Ingredient Search ('Chicken'):

SELECT * FROM pizza_types
WHERE name LIKE '%Chicken%';

SELECT 
  ROUND(AVG(COALESCE(price,0)),2)

Phase 3: Revenue Analysis

Total Order Value (Join + Group By):

SELECT
  od.order_id,
  SUM(od.quantity * p.price) as total
FROM order_details od
LEFT JOIN pizzas p 
  ON od.pizza_id = p.pizza_id
GROUP BY od.order_id;

Phase 3: Category Insights

Multi-table joins to find top performers.

SELECT pt.category, SUM(od.quantity) as total_qty
FROM order_details od
JOIN pizzas p ON od.pizza_id = p.pizza_id
JOIN pizza_types pt ON p.pizza_type_id = pt.pizza_type_id
GROUP BY pt.category
HAVING SUM(od.quantity) > 5000;

Phase 3: Inventory Insights

Dead Stock Detection

Identifying pizzas on the menu that have never been ordered.

SELECT p.pizza_id, p.size
FROM pizzas p
LEFT JOIN order_details od
  ON p.pizza_id = od.pizza_id
WHERE od.order_id IS NULL;

Business Value: Remove from menu to reduce waste.

Phase 3: Price Gap Analysis

Self-Join Strategy:

Comparing the price difference between sizes (e.g. L vs M) for the same pizza.

SELECT 
  p1.size, p1.price, 
  p2.size, p2.price,
  (p2.price - p1.price) as diff
FROM pizzas p1 
JOIN pizzas p2 
  ON p1.pizza_type_id = p2.pizza_type_id
  AND p1.size != p2.size;

Strategic Recommendations

Eliminate Waste

Discontinue the "Dead Stock" items identified in Slide 9 to reduce inventory costs and streamline the menu.

Drive Upsells

Leverage the significant price gap (Slide 10) to incentivize customers to upgrade to Large pizzas for better value.

Optimize Staffing

Align staff schedules with the peak order hours (12-1pm & 6-7pm) identified in Phase 2 to reduce wait times.
